7 Settings

To configure the Ortho System settings, click in the patient browser.

 

 

 

Note: If you change any settings on the Settings page and want them to take effect in the relevant application (i.e. Ortho Analyzer or Appliance Designer), they need to be saved by clicking the Save  icon and the relevant application needs to be restarted.

 

 

Under System settings, configure the settings for:

 

   

OrthoData

 

Allows you to configure the Ortho data settings.

 

System

Option

Description

Decimate input models to

Set the number of triangles in input models after preparation to decimate the models by default.

Save models copy to STL

Select to save models in the .stl format together with the .dcm format when models are being scanned or exported.

Use autosave during modeling

Select to set the frequency at which the model will be automatically saved during the workflow.

Disable AutoSegmentation

Select to prevent the auto-segmentation and do the manual segmentation instead.

Use 2D tooth length

Select to use the traditional two-dimensional measuring system which enables measuring dental distances using the user's point of view as a measuring plane.

Restrict some calculations to one CPU core

Select to restrict Appliance Designer performance by using one CPU core.

Large preview

Select to display large previews in the applications.

 
  Export

Option

Description

Default export folder

Choose the desired folder for saving exported models. Click Browse to select the path.

Export format

Choose the format for the exported files:

  • .stl - a standard open format.
  • .dcm - a 3Shape compressed format which gives smaller files but is not supported by external software.

Use Qualified Names

Select to uniquely name the exported model sets, e.g. based on the patient and model set IDs.  Clear to give the exported model sets their default names, Maxillary.xxx and Mandibular.xxx correspondingly.

Export TRIOS models into Zip files

Select to export TRIOS models without patched areas to .zip files.

Option

Description

Export format

Choose the format for the exported files:

  • .stl - a standard open format.
  • .dcm - a 3Shape compressed format which gives smaller files but is not supported by external software.

Folder format

Choose the desired folder for saving exported models. Click Arrow to specify the folder path. By default, the exported models are saved to PatientFolder/ModelsFolder/ModelSetFolder.

Name format

Define a default name for the models exported from Virtual Setup.

Save export models to PLY/OBJ

Select to save the exported models in the .ply and .obj formats in addition to the .stl format.

Save teeth and gingiva separately to PLY/OBJ

Select to save the exported teeth and gingiva separately.

Include production relevant OBJs

Select to generate the production files in the .ply and .obj formats.

Do not use unreadable symbols (line breaks, tabulation, etc.) in production files

Select to prohibit unreadable symbols (such as line breaks, tabulation, etc.) in the production file names.

Export Production Files Overview (ClearAlginerProductionrResources.xml)

Select to export ClearAlginerProductionrResources.xml file (enabled by default).
   Export in background

 

Allows you to enable and set up time-out for export in the background.

 

 

Note: These parameters apply to the export performed using the Export Model As option in the Appliance Designer software, and also to the Export options in the Ortho System patient browser (see the chapter Importing Scans).

 

Note: To export study models as .stl, ensure you have an active license of the STL Study Model Builder. To export planning models as .stl, ensure you have an active license of the STL Planning Model Builder.

 

Note: To save Virtual Setup and Retainers export models in the .ply and .obj formats, activate a corresponding license first. Contact your 3Shape reseller for details.
