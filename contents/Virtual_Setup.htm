The Treatment step provides options to simulate orthodontic treatment by moving the teeth and placing the brackets virtually.

 

The Treatment tab contains the relevant options described in the following.

 

Note: Clicking one of the options in the Treatment tab opens the License Terms window with IMPORTANT INFORMATION AND LEGAL DISCLAIMER, also shown below. Please read the information carefully before proceeding.

 

Important information and legal disclaimer

 

Virtual Setup is an option simulating the orthodontic treatment in Ortho Analyzer.

 

The Virtual Setup option allows you to create different setups and subsetups with the teeth being extracted, moved etc., compare the created setups when desired and obtain a precise summary of all teeth movements.

 

The movements for each tooth are initially presented in the Timeline view (1).

  • Go to the needed subsetup by double-clicking the grey-gradient line between the subsetup numbers that represents the subsetups (2) - the current subsetup column is marked with dark grey color in the view and applied on the model.
  • By default, movements are represented as Combined (3), i.e. for all types of movements jointly, - to distinguish between different movements (e.g. inclination, extrusion/intrusion) by means of color, select the corresponding movement highlighter. Similarly, enable the visualization of stripping and/or attachment to see at which subsetups they are used.
  • Adjust the breakdown of movements within a setup by selecting a tooth or multiple teeth while holding down Ctrl/Shift and dragging the edge to the needed point on the timeline (4).

  • The Optimize movements option (5) becomes available if you enable per-aligner constraints (see Per-aligner constraints below). Use this option to optimize the number of subsetups needed to achieve the final tooth position based on the selected per-aligner constraints.

  • Collisions button (6)  - allows you to detect:

    		  - the intersections between tooth models (neighboring teeth).

    		 - intersection with the value lower than the threshold value.

Hover over collision icons listed above to see the number of the tooth and collision value compared to the threshold value.

 

Adjusting teeth movements in the Timeline view

 

  • Select a single tooth to go to the Way-points view (7) showing the tooth's movements at each subsetup. You can continue adjusting teeth position by moving them on the model, or you can use the points in the view.

  • Right-click a point on the graph to choose from available options (8). You can also apply these options to multiple points (selected while holding down Ctrl/Shift).

Lock (Alt+Click) - allows you to set way-points, i.e. specific positions of the tooth at particular subsetups, for further adjustment of the whole tooth movement according to those locked positions, marked by a lock icon (9).  

  • The Adjust to way-point option (10) accommodates the tooth movement at all subsetups according to the way-points, i.e. locked positions. All manually performed changes to the tooth position on the model are preserved.
  • Undo/redo your changes using the corresponding buttons (11).

Movements - allows you to rotate and move teeth as desired. To select the tooth, click on it with the mouse. The selected one is highlighted. Double-click the tooth to activate the control points. When active, control points turn white.

 

Option Description

Perform collision detection

Activate this option to analyze the intersections between tooth models (neighboring teeth). The collision detection is calculating the maximum overlap between two teeth models. This overlap is displayed with a red segment visible when the teeth are set as transparent and the corresponding setups/subsetups are marked with (if the collision value is higher than the threshold value) or (if the collision value is lower than the threshold value) in the tree view. In addition, this option displays the distance between teeth.

Restrict teeth collisions

Use this option to limit the movement of the selected tooth according to the defined collision threshold value.

Use collision threshold

Select it to activate the collision threshold value. Intersections with values lower than the defined threshold will be displayed in grey and the ones above the threshold will be marked in red.

Collision threshold value

Allows you to indicate the collision threshold value manually.

Show collision values

This option displays accurate intersection values at any selected tooth showing the level of penetration in millimeters.

Band Size Calculation

If selected, calculates the size of the band and displays it on the screen.

Ideal arch 

Click to activate the corresponding drop-down list, where you can select the desired arch to move the teeth accordingly. You can use the one created using the Dental arch analysis option with the 2D overlays (see the chapter Analysis).

Per-aligner constraints

Click to activate the corresponding drop-down list, where you can select the desired constraints to move the teeth accordingly. These constraints can be associated with mechanical properties of aligners or with the amount of prescribed teeth movement per aligner and can be set on the Settings page (see the chapter Settings). If the constraints are violated, the corresponding window appears, notifying where such violation occurred and asking if you want to correct the teeth positions according to the applied constraints.

Teeth constraints 

Teeth Constraints option is turned on by default. When you create a new setup, a slide-in-message notifying of the default constraints activation appears. The constraints placed at the top of the drop-down list on the Settings page are set as default in Virtual Setup. To change the default teeth constrains, move the desired constraints to the top of the list on the Settings page. You can also select the teeth movement constraints from the corresponding list in Virtual Setup. These constraints can be associated with tooth biomechanical and morphological rules or to the mechanical properties of selected appliances and can be set on the Settings page (see the chapter Settings). If the constraints are violated, the corresponding notification appears.

Extract teeth - allows you to extract the selected tooth on the mandible or maxilla.

 

Stripping - allows you to simulate the interproximal reductions (IPR) for the selected tooth according to the parameters you enter.

 

Note: If there are teeth intersection of more than 0.1 mm, they will be marked in the IPR Report & Attachments section of the generated report as those which require manual stripping (see the chapter Analysis).

 

Note: Only the automatic and parametric stripping values can be summarized in the Teeth movements overview table. The manual stripping will also be shown in the table, but without values.

 

Attachments - allows you to place attachments on the selected tooth. Apply the Automatic placement option to a selected tooth for the relevant attachment from the chosen attachment group to be placed automatically. The priority of the automatic placement of attachments within a group, matched by movement conditions, is defined by their order on the Settings page. In case the teeth movements fall under the rules set for several attachments, the one being higher on the list will be applied.

 

Note: If the Enable automatic placement checkbox is selected for at least one of the attachments in the group, the whole group will be displayed under the Automatic Placement option in Virtual Setup.

 

Edit flexible tissue area - allows to change the borders of the tissue area. Click the Show flexible area button to view the splines.

 

Export Setups - exports setups to the selected folder. In the Save as... window, you can fold/unfold the groups and select which models or model sets you wish to export.

 

 

Note: The export of planning models (virtual setups) as .stl files requires an active license of Planning Model Builder.

 

Choose from available options when exporting virtual setups:

  • Export models to files - selects all upper and lower models for export as separate files.
  • Export attachment templates - activates additional subsetups with attachment(s) and allows you to enter additional molar tag.
  • Export models for retainers - allows to generate one or several models for the retainer that is used to maintain the correct occlusion after the treatment is completed.
  • Save model sets to database - selects all model sets for export as model sets to the patient browser.
  • Export for printing - allows changing the position of the model so that z-axis follows the teeth direction.
  • Merge attachments with models - merges attachments with the surface of models.
  • Open mouth slider - allows to preview upper and lower jaws in the open/closed position.

 

Note: When exporting a setup, the last unchanged jaws (upper or lower models), as compared to the previous subsetup, will not be selected for export in the Save as... window so that you don't export and further produce identical models. This results in a different number of exported models per jaw. For example, if there are three subsetups in a setup sequence where all three have been changed for the lower jaw and only the second one has been changed for the upper jaw, then the third subsetup model will not be selected for import for the upper jaw. As a result, all three models will be exported for the lower jaw while for the upper jaw, only two models will be exported as the third one will not be selected for no change.

 

  • ID Tags - allows adding ID tags on the surface of the model. Input text and/or choose an attribute from the available list by clicking . Move the tag on the models using the arrow buttons.

 

Note: The position of an ID tag is preserved in relation to the given fixture between different cases.

 

  • Molars - allows adding the tags on molars so that the resulted model is marked respectively for better distinction. These tags are set up similarly to ID tags.

 

Note: When using the ID Tags or Molars option, if you decide not to export all subsetups in a setup, then the subsetups that are actually exported will be tagged by subsetup number in an unbroken sequential order. For example, if there are 5 subsetups, and you choose not to export number 3, then the remaining exported subsetups with be tagged as 1, 2, 3 and 4, respectively, in the correct sequence.

 

  • Marking Tag Simulation -  allows you to export the position of the tag for laser engraving. As a result of export, an .svg file is generated with necessary information for engraving the tag on the model, along with the file containing the jaw model.

  • Graphics Marking Tag Simulation - allows identifying an area for boundaries that could contain the graphics tag. After the export, the information on the graphics marking tag is written to an SVG file along with the corresponding information on the marking tag.

  • Export Production Files -  during the export, the additional file containing the information about the exported files is generated and saved to the Models folder of a current patient case.

  • Cut-Spline -  generates the cutout spline for each exported setup when both Export for printing and Cut-spline options are enabled. You can set such parameters as Offset, Wave offset and Aligner termination type. The cut-spline output is generated in .dxf format.

  • Hollowing and wax-up - select to use relevant options in Mold Adjustment.

 

Note: In order to use Marking Tag Simulation, Graphics Marking Tag Simulation, Export Production Files and Cut-Spline1 options, ensure that the appropriate licenses are enabled on your dongle. For more details, contact your reseller. (1: Please contact your local sales representative for availability.)

 

If you want to export a case while you are still working in Ortho Analyzer, use the Export in background option. Cases exported using this option are put into an export queue for execution. To open the export queue, click   in the patient browser. Here you can see the of cases being exported with “In progress” status.

 

After the export is completed, you can find the files in the Exports section of the corresponding patient / model set. From here you can access the destination folder where the exported files are saved, and open cases in Mold Adjustment. You can also work with the exported cases on different PCs within the same Client-Server setup. Refer to the Technical Documentation for more details.
